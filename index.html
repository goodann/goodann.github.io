컴퓨터 소프트웨어 개발자

자기소개

중학교 3학년때 Visual Basic을 독학하여 프로그래밍을 처음 접하였고, 흥미를 느껴 이때부터 장래희망을 프로그래머로 정했습니다.
그 때 공부를 검색과 다른 사람의 코드를 수정해보는 등의 방법으로 빠르게 문법과 기능에 대해 흡수하였으며, 프로그래밍을 공부하기 위해 실업계 고등학교로 진학을 하였습니다.
고등학교에 들어 C언어를 접하였고, 경기도 실업계 고등학교 경진대회 프로그래밍 부분에서 은상을 수상하기도 하였습니다.
그 이후에 광운대학교 진학, 컴퓨터소프트웨어학과로 진학하여 프로그래밍을 비롯한 컴퓨터 과학을 습득하였습니다.
학부시절에 후배들을 위해 프로그래밍 스터디를 여러 번 진행 하였으며, 전공과목들의 과제를 함에 있어 막힘이 없이 거의 모든 과제를 기간 내에 해내었고,
C/C++/JAVA/C#/Python등 여러 프로그래밍 언어를 공부하였고, 언어별로 과제나 프로그램 등을 만들었습니다. 그 외에도 개인적으로 필요한 툴 들을 제작하기 위해 MFC로 작성된 코드들을 검색하여 자신만의 프로그램을 제작 해보았으며 인터넷 검색 등으로 찾은 정보와, 다른 사람이 공개한 라이브러리 코드를 참고하고 수정하여 작성된 것들이 많습니다.
졸업 후 매크로그래프 회사에 입사하여, C++, Qt, OpenCV, OpenGL을 이용하여 GUI프로그램을 제작하였으며 게임엔진인 Unity엔진의 Plugin을 짜는등의 경험도 해 보았습니다.
매크로그래프에서 1년 8개월 정도 생활을 한 다음, 게임회사인 펄어비스로 이직하여 C++, MSSQL, Lua를 이용하여 라이브 서비스 중인 게임의 새로운 컨텐츠를 개발을 하였습니다. 8개월 간 개발을 한 다음 퇴사를 하여 현재에 이르렀습니다.
저의 장점은 창의적인 생각으로 좋은 아이디어를 내는 점과, 평소에도 당연한 것을 다른 방식으로 해결하면 어떨까 라는 생각을 합니다.
그러한 생각으로 더 빠르게 과제를 해결하거나 어디를 가던 미리 경로를 찾을 때 여러 가지 경로를 찾아 출발할 때 다시 점검해서 가장 경로를 찾기도 하며, 팀 프로젝트에서 결정적인 생각을 제공하여 팀의 작업속도를 빠르게 한 적도 있습니다.
또한 맡은 일은 반드시 책임지고자 하며, 이러한 책임감으로 팀 프로젝트 중에 한 명이 마지막에 연락이 안되어 위기에 빠진 적이 있지만, 그 사람의 부분까지 제가 다 진행하여 팀 프로젝트를 완성시킨 적도 있습니다.

프로젝트

테일즈위버 딜레이 계산기 + 메모리 최적화 및 타이머 프로그램

사용기술 : C++, STL, MFC

테일즈 위버 게임을 하는 중, 일정메모리량 이상의 메모리를 테일즈 위버가 사용할 경우 테일즈위버가 종료되는 현상이 커뮤니티를 중심으로 발견되어 이를 해결하려고 주기적으로 테일즈위버의 메모리 사용량을 줄이는 프로그램을 제작, 그 프로그램에 게임에 필요한 딜레이 계산, 보스 재입장 타이머 기능 등 부가기능을 추가하는 프로그램을 제작하였다.

마인크래프트 bukkit 서버 관리 프로그램

사용기술 : C++, MFC

마인크래프트 Bukkit을 이용하여 마인크래프트 서버를 열 때, 명령어 입력, 서버 자동 재시작, 서버가 이용할 메모리 설정 로그 저장 등 몇몇 설정을 편하게 하기 위해 제작한 프로그램, bukkit프로세스를 실행하여 그 프로그램의 stdin/stdout을 MFC를 이용하여 GUI상에서 입/출력이 가능하도록 리다이렉션 하였다.




Unity를 이용한 레이싱 게임

사용기술 : C#, Unity
역할 : 아이디어 기획, 플레이어(자동차)움직임 제작, 파티클 이펙트 제작

졸업 작품으로 멀티플레이를 지원하는 물리엔진을 이용한 게임 제작을 진행함 팀 인원은 4명으로, Unity5를 이용한 레이싱 게임 제작하였다. 플레이어 자동차의, 부분 파괴를 구현하여 달리면서 실시간으로 플레이어 및 상대의 부위가 파괴는 것을 실시간으로 볼 수 있었다. 게임의 주요 포인트는 방해물과 충돌로 오프로드 레이싱을 기반으로 만들었다. 카트라이더의 부스터 시스템을 차용하여, 드리프트+부스터가 존재하였고, 맵 중간중간에 파괴 가능한 방해물들을 배치하여 이를 부술 수 있는 미사일을 제공하여 이를 해쳐나가면서 레이싱을 하는 기획으로 진행하였다.

Kinect 4대를 이용한 현실 물체를 3D공간상에 복원

사용기술 : C++, OpenCV, OpenGL, libfreenect

Kinect 4대에서 얻은 이미지를 이용하여 3D공간상에 복원하는 기존 시스템을 이용하여, 실시간으로 Kinect에서 데이터를 얻어와 3D공간 상으로 복원시켜 OpenGL을 이용하여 실시간 렌더링 하는 프로그램을 제작.이후 오큘러스 리프트에 적용 가능하도록 확장하였다.

스테레오 카메라를 이용한 3D Face Capture

사용기술 : C++, Qt5, OpenGL, OpenCV, nodeeditor

프롬프트 기반의 여러 프로그램으로 나뉘어진 3D얼굴 복원 시스템을 NodeBase의 GUI프로그램으로 이식하여 원하는 프로세스를 원하는 순서대로 진행가능하도록 제작하였다.
연속된 시퀀스 이미지를 불러와서 해당 이미지상의 같은 지점이 이전/다음 이미지에서 어디에 위치하는지 추적하는 프로그램을 제작하였다. OpenCV의 기능을 이용하여 자동적으로 추적하는 기능읋 넣었고, 이를 보완하여 사용자가 수동으로 추적 하는기능을 추가하였고, 깊이 이미지와 카메라 파라미터가 있으면 3D공간상으로도 점 추적을 볼 수 있도록 추가하였다.
복원된 Mesh를 표준모델로 변환할때 사용되는 프로그램 Wrap3D에서 사용하는 Point데이터를 여러 시퀀스 이미지에서 Point Tracking에서 사용할 수 있도록 2D-3D간 Point Projection하는 프로그램 개발하여 위의 프로그램과 연동하여, Wrap3D프로그램에서 여러 이미지에서 복원을 좀더 용이하게 하도록 프로그램을 제작하였다.

홀로그램 제작을 위한 데이터 정제 및 씬 세팅을 위한 Unity3D 플러그인 제작

사용기술: C++, C#, Unity

Maya에서 만든 Mesh를 유니티 씬에 좌표계를 변환하여 원하는데로 입력되도록 작업하였다.
Point Cloud 데이터를 유니티 씬에 입력되도록 제작하였다. 유니티에서 기본적으로 Points를 그리는 기능이 없어서 버텍스 데이터만 존재하는 데이터를 보여주도록 셰이더를 만들었다.
세팅된 씬 데이터를 홀로그램 제작을 위한 input Data(XML)로 정제하여 출력하도록 플러그인 작성하였다.





검은사막 컨텐츠 개발

사용기술: C++, 서버, 클라이언트, Lua, UI, SQL

PC/Console에서 서비스 중인 온라인 MMORPG 검은사막의 컨텐츠를 기획에 맞추어 구현하는 역할을 하였다.
검은사막 기술창 리뉴얼에 중반부 이후 대부분의 클라이언트 및 UI스크립트 작업을 하였다.
검은사막 모험일지의 추가기능과 알림 관련하여 클라이언트 및 UI스크립트 작업을 하였다.
시즌서버 보상을 수령하는데 있어 관련된 서버, 클라이언트, DB, UI 작업을 진행하였다.
섀도우아레나 말풍선 UI 작업을 하였다.
기타 버그 수정들을 했다.
